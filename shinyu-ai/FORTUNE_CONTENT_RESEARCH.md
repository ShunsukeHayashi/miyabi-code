## プロジェクト概要

### コンセプト

**誕生日・名前・出生地/時間・兄弟構成を入力するだけで、世界中のあらゆる占いを統合して総合プロファイルを生成する革新的な占いアプリ**

- 生年月日（必須）
- 出生時刻（推奨、ホロスコープ精度向上）

- 統合プロファイル（2000-3000文字の自然な日本語）
- 西洋占星術チャート（ホロスコープ図）

- 兄弟構成診断
- ユーモア系占い（野菜・キノコ・フルーツ・天気）
- 時系列分析（過去10年・現在・未来10年）
- AI生成画像（各カテゴリ）

**調査深度**: ⭐⭐⭐⭐⭐
**データ品質**: ⭐⭐⭐⭐⭐

#### 調査内容

**12動物分類**:
1. 狼: 一匹狼タイプ
2. 猿: 明るく社交的
3. 虎: リーダーシップ
4. 子守熊（コアラ）: マイペース
5. 黒豹: クールでミステリアス
6. ライオン: 王者の風格
7. チータ: スピード感
8. ペガサス: 自由を愛する
9. 象: 安定感
10. たぬき: 世渡り上手
11. こじか: 純粋で素直
12. ひつじ: 温和で従順

**60分類**: 12動物 × 5カラー（ブルー、グリーン、レッド、イエロー、ブラウン）

各カラーは性格の強弱や傾向の違いを表現

#### 計算アルゴリズム

**方法1: シリアル値計算**
```
キャラクター番号 = ((シリアル値 + 8) % 60) + 1

シリアル値 = 生年月日のExcelシリアル値
           （1900-01-01を1とする日数）
```

**方法2: 早見表方式**
1. 生まれた年と月が交わる数字を早見表から取得
2. その数字に生まれた日を足す
3. 合計が61以上なら60を引く
4. 結果の1-60の番号で動物キャラを特定

#### データソース

- 動物占い公式サイト

### 3. 数秘術（Numerology）

**調査深度**: ⭐⭐⭐⭐⭐
**データ品質**: ⭐⭐⭐⭐⭐

#### 調査内容

**運命数（Life Path Number）**: 1-9 + マスターナンバー11, 22, 33

各数字の意味:
- 1: リーダー・パイオニア
- 2: 調和者・外交官
- 3: 表現者・クリエイター
- 4: 建設者・実務家
- 5: 冒険者・自由人
- 6: 養育者・責任者
- 7: 探求者・哲学者
- 8: 達成者・権力者
- 9: 人道主義者・完成者
- 11: インスピレーションの使者（マスターナンバー）
- 22: マスタービルダー（マスターナンバー）
- 33: マスターティーチャー（マスターナンバー）

**表現数（Expression Number）**: 名前から算出される才能・表現方法

#### 計算アルゴリズム

**運命数**:
```
1. 生年月日の全桁を1桁ずつ足す
   例: 1988-06-11 → 1+9+8+8+6+1+1 = 34
2. 合計が11, 22, 33の場合はそのまま（マスターナンバー）
3. それ以外は1桁になるまで足す
   例: 34 → 3+4 = 7
```

**表現数**:
```
1. A=1, B=2, ... Z=26 でアルファベットを数値化
2. 名前のすべての文字を数値化して合計
3. マスターナンバー（11, 22, 33）チェック
4. それ以外は1桁になるまで足す
```

#### データソース

- カバラ数秘術研究文献
- Numerology専門書（Hans Decoz等）

### 4. 兄弟構成診断（Sibling Order Psychology）

**調査深度**: ⭐⭐⭐⭐⭐
**データ品質**: ⭐⭐⭐⭐

#### 調査内容

**4つの出生順位**:

1. **長子（第一子）**
   - 責任感が強い
   - リーダーシップがある
   - 完璧主義の傾向
   - 親から高い期待

2. **中間子（真ん中の子）**
   - 柔軟で適応力がある
   - 平和主義で調停役
   - 独自性を求める
   - 注目を集めにくい環境から独立心

3. **末っ子**
   - 明るく社交的
   - 甘え上手で愛されキャラ
   - 自由奔放で創造的
   - 常に年上に囲まれる環境

4. **一人っ子**
   - 成熟していて大人びている
   - 独立心が強い
   - 完璧主義で几帳面
   - 大人との関わりが多い

#### 影響要因

**性別**: 長男/長女、末っ子男/女で微妙に異なる

**年齢差**:
- 0-2歳差: 競争が激しい
- 3-5歳差: 標準的、バランス
- 6歳以上: 一人っ子的要素

**兄弟人数**: 2人、3人、4人以上で役割が変化

#### 理論的基盤

- アドラー心理学の出生順位理論
- 日本の心理学研究（文教大学、北海道女子短期大学等）
- 近年の研究: 成長と共に影響は薄れる傾向

#### データソース

- 文教大学 出生順位研究論文（2009）

### 6. 時系列分析（Temporal Analysis）

**調査深度**: ⭐⭐⭐⭐⭐
**データ品質**: ⭐⭐⭐⭐⭐

#### 手法: トランジット法（Transit）

現在の惑星位置が、出生図（ネイタルチャート）の惑星とどのようなアスペクトを形成するかを分析

#### 主要サイクル

1. **木星リターン**: 約12年周期
   - 新たな成長と拡大の時期
   - 幸運と機会が訪れる

2. **土星リターン**: 約30年周期
   - 人生の節目（29歳、59歳）
   - 責任と成熟を求められる試練
   - 新しいステージへの移行

3. **天王星オポジション**: 約84年の半周期（42歳）
   - 中年の危機
   - 人生の大きな転換点

4. **海王星スクエア**: 約41歳
   - 精神的な目覚めや混乱
   - 現実と理想のギャップ

#### 分析期間

- **過去10年**: 経験した出来事の意味を解釈
- **現在（±1年）**: 今年のテーマと月ごとの運勢
- **未来10年**: 今後の成長の方向性、転換期の予測

#### データソース

- Transit Chart Calculator（複数サイト）

### 7. 統合プロファイル生成アルゴリズム

**調査深度**: ⭐⭐⭐⭐⭐
**データ品質**: ⭐⭐⭐⭐

#### アルゴリズムフロー

```
1. 各占いシステムから結果を取得
   ├─ 西洋占星術
   ├─ 動物占い
   ├─ 数秘術
   ├─ 兄弟構成
   └─ ユーモア系（オプション）

2. 共通特性を抽出
   - キーワードマッチング
   - 意味的類似性分析
   - 2つ以上の占いで出現した特性を抽出

3. 矛盾を調整
   - 矛盾は個人の多面性として肯定的に捉える
   - 状況による使い分けとして説明
   - 例: "社交的だが一人も好き" → "ON/OFF切り替え上手"

4. GPT-4でプロファイル文章生成
   - プロンプト構築
   - 2000-3000文字の自然な日本語
   - 温かく励ます口調

5. 出力
   - 統合プロファイル本文
   - 個別占い詳細
   - 共通特性リスト
   - 時系列分析
   - ビジュアル要素（画像URL）
```

#### ウェイト配分

各占いシステムの信頼性と重要度に基づくウェイト:

- 西洋占星術: 35%（最も包括的で詳細）
- 数秘術: 25%（深い洞察）
- 動物占い: 20%（相性診断）
- 兄弟構成: 15%（環境要因）
- ユーモア系: 5%（エンターテイメント）

**注**: 出生時刻・出生地がない場合は西洋占星術のウェイトを下げる

#### プロファイル構成

1. **イントロダクション**（150-200文字）
   - 総合的な第一印象
   - コアな性格特性

2. **詳細な性格分析**（400-600文字）
   - 基本性格（西洋占星術・数秘術）
   - 行動パターン（動物占い）
   - 対人関係の傾向（兄弟構成）

3. **強みと才能**（200-300文字）
   - 共通して現れる強み
   - 箇条書き + 説明

4. **課題と成長の方向性**（200-300文字）
   - 弱み・課題
   - ポジティブで励ます表現

5. **恋愛傾向**（200-300文字）
   - パートナーシップ

6. **仕事適性**（200-300文字）
   - キャリアパス

7. **人生の旅路**（300-500文字）
   - 過去10年の振り返り

   - 未来10年の展望

8. **ユーモア要素**（100-150文字）
   - 野菜・キノコ・フルーツ占い

## 技術スタック推奨

### バックエンド

**推奨**: Python + FastAPI

**理由**:
- Swiss Ephemeris の Python binding（pyswisseph）が優秀
- Kerykeion などの占星術ライブラリが豊富
- FastAPI は高速で OpenAPI ドキュメント自動生成
- OpenAI SDK が公式サポート

**代替案**: Rust + Actix-web（高パフォーマンス重視）

### フロントエンド

**推奨**: Next.js 14 + TypeScript

**理由**:
- SSR/SSG による SEO 最適化
- Vercel へのデプロイが簡単
- TypeScript で型安全性確保
- React エコシステムが豊富

### データベース

**推奨**: PostgreSQL + Redis

**理由**:
- PostgreSQL: JSONB型で柔軟なデータ保存
- Redis: 計算結果のキャッシュ、セッション管理

### AI統合

**推奨**: OpenAI GPT-4 + DALL-E 3

**理由**:
- GPT-4: 最高品質の文章生成
- DALL-E 3: 簡単なプロンプトで高品質画像
