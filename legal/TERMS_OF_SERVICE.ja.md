## 第1条 定義

本規約において、以下の用語は以下の意味を有します。

1. **「本サービス」**: Shunsuke Hayashiが提供する「Miyabi」という名称の完全自律型AI開発オペレーションプラットフォーム（オープンソース版、SaaS版、Web Platform版を含む）を指します。

2. **「当社」**: Shunsuke Hayashiを指します。

3. **「ユーザー」**: 本規約に同意の上、本サービスを利用する個人または法人を指します。

4. **「アカウント」**: 本サービスを利用するためにユーザーが作成する登録情報を指します。

5. **「AI生成コード」**: 本サービスがAnthropic社のClaude AIを使用して自動生成したソースコード、テストコード、ドキュメント等を指します。

6. **「コンテンツ」**: ユーザーが本サービスに投稿・保存するコード、データ、Issue、Pull Request等の情報を指します。

7. **「オープンソース版」**: Apache 2.0ライセンスの下で公開されている本サービスのソフトウェアを指します。

8. **「SaaS版」**: 当社が提供する有料のクラウドサービスとしての本サービスを指します。

9. **「料金プラン」**: Free、Pro、Team等、本サービスの利用条件と料金を定めたプランを指します。

## 第3条 サービス内容

### 3.1 提供するサービス
本サービスは、以下の機能を提供します。

1. **自律型AI開発自動化**
   - Issue作成からコード実装、PR作成、デプロイまでの完全自動化
   - Claude AI（Anthropic）によるAIコード生成

2. **7つのCoding Agents**
   - CoordinatorAgent（タスク統括）
   - IssueAgent（Issue分析）
   - CodeGenAgent（コード生成）
   - ReviewAgent（品質レビュー）
   - PRAgent（PR作成）
   - DeploymentAgent（デプロイ）
   - TestAgent（テスト）

3. **GitHub OS統合**
   - GitHub Issues、Actions、Projects V2等との統合
   - GitHub OAuth 2.0認証

4. **ナレッジ管理システム**
   - Vector DB（Qdrant）によるログ検索
   - 過去の実行履歴の学習

### 3.2 サービスの変更
当社は、ユーザーへの事前通知なしに、本サービスの内容を変更、追加、削除することができます。ただし、重要な機能の削減や有料プランの価格変更については、第17条に従い事前通知を行います。

### 3.3 料金プラン

#### 3.3.1 Free プラン
- **価格**: $0/月
- **対象**: 個人開発者、オープンソースプロジェクト
- **機能**:
  - 月間Issue処理: 100件まで
  - Claude API使用量: 月10,000トークンまで
  - 並列Agent実行: 1並列
  - Public リポジトリのみ対応
  - サポート: Community（GitHub Discussions）
  - SLA保証: なし

#### 3.3.2 Pro プラン
- **価格**: $29/月（年払い$290、月額換算$24.17）
- **対象**: フリーランス、スタートアップ（1-5人）
- **機能**:
  - 無制限 Issue処理
  - Claude API使用量: 月100,000トークンまで
  - Private リポジトリ対応
  - 並列Agent実行: 3並列
  - 優先サポート（Email、24時間以内返信）
  - SLA保証: 99.0%
  - ダッシュボード（リアルタイムメトリクス）

#### 3.3.3 Team プラン
- **価格**: $99/月/5ユーザー（年払い$990、月額換算$82.50）
- **対象**: 成長企業（6-50人）
- **機能**:
  - Proの全機能
  - Claude API使用量: 月500,000トークンまで
  - 並列Agent実行: 10並列
  - SSO/SAML認証
  - 優先サポート（Email、12時間以内返信）

## 第7条 AI生成コードに関する免責・ユーザー責任 ★最重要

### ⚠️ 本条は極めて重要です。必ずお読みください。

### 7.1 AI生成コードの性質

#### 7.1.1 AIによる自動生成
本サービスは、Anthropic社のClaude AIを使用してソースコードを自動生成します。AI生成コードは、人間が作成したコードと同様に、以下のリスクを含む可能性があります。

1. **エラー・バグ**: 構文エラー、論理エラー、ランタイムエラー等
2. **セキュリティ脆弱性**: SQL Injection、XSS、CSRF、情報漏洩等
3. **パフォーマンス問題**: メモリリーク、無限ループ、過度なリソース消費等
4. **互換性問題**: ライブラリバージョンの不一致、API仕様の変更等
5. **法令違反**: 著作権侵害、特許侵害、ライセンス違反等

#### 7.1.2 品質保証なし
当社は、AI生成コードの品質、正確性、完全性、適時性、セキュリティについて、いかなる保証も行いません。AI生成コードは「**現状有姿（AS IS）**」で提供されます。

#### 7.1.3 予測不可能性
AIモデルは確率的に動作するため、同じ入力に対しても異なる出力を生成する可能性があります。また、AIモデルの学習データに基づき、既存のコードパターンを再現する可能性があります。

### 7.2 ユーザーの必須責任

#### 7.2.1 生成コードの必須レビュー義務 ★
ユーザーは、AI生成コードを本番環境にデプロイする前に、**必ず以下のレビューを実施する義務を負います**。

1. **コードレビュー**: 全てのコードを人間が確認すること
2. **セキュリティレビュー**: セキュリティ脆弱性の有無を確認すること
3. **ライセンスレビュー**: 使用しているライブラリのライセンスを確認すること
4. **法令遵守確認**: 生成されたコードが法令に違反していないことを確認すること

#### 7.2.2 テスト実施義務 ★
ユーザーは、AI生成コードを本番環境にデプロイする前に、**必ず以下のテストを実施する義務を負います**。

1. **単体テスト**: 各関数・モジュールの動作確認
2. **統合テスト**: システム全体の動作確認
3. **セキュリティテスト**: 脆弱性スキャン、ペネトレーションテスト等
4. **パフォーマンステスト**: 負荷テスト、ストレステスト等
5. **本番環境以外でのテスト**: ステージング環境、テスト環境等での十分なテスト

#### 7.2.3 セキュリティ検証義務 ★
ユーザーは、AI生成コードのセキュリティを検証する責任を負います。具体的には、以下の対策を実施することを強く推奨します。

1. 静的解析ツール（ESLint、TypeScript、CodeQL等）の使用
2. セキュリティスキャンツール（Gitleaks、Snyk等）の使用
3. 依存関係の脆弱性チェック（npm audit、cargo audit等）
4. 外部セキュリティ専門家によるレビュー（重要システムの場合）

#### 7.2.4 本番環境デプロイの責任 ★
**ユーザーは、AI生成コードを本番環境にデプロイする最終判断と責任を負います。**

当社は、以下について一切関与せず、責任を負いません。

1. 本番環境へのデプロイの可否判断
2. デプロイ後の監視・運用
3. データ損失・システム障害の防止
4. ロールバック・復旧作業

### 7.3 当社の免責事項

#### 7.3.1 AI生成コードの品質
当社は、AI生成コードの以下について、いかなる保証も行わず、責任を負いません。

1. 品質・正確性・完全性
2. セキュリティ・安全性
3. 商品性・特定目的適合性
4. 第三者の権利非侵害

#### 7.3.2 セキュリティ脆弱性
AI生成コードにセキュリティ脆弱性が含まれていた場合でも、当社は責任を負いません。ユーザーは、第7.2.3条に従い、自らセキュリティ検証を実施する責任を負います。

#### 7.3.3 データ損失・システム障害
AI生成コードに起因する以下の事象について、当社は責任を負いません。

1. データの損失・破損・漏洩
2. システムの停止・遅延・障害
3. データベースの破損
4. バックアップの失敗
5. セキュリティインシデント

#### 7.3.4 間接損害・逸失利益
AI生成コードに起因する以下の損害について、当社は責任を負いません。

1. **逸失利益**: 事業機会の損失、売上の減少等

3. **特別損害**: 予見可能性の有無を問わない
4. **派生的損害**: 二次的・三次的に生じた損害

#### 7.3.5 第三者からのクレーム
AI生成コードに関して第三者（顧客、エンドユーザー、権利者等）からクレーム・訴訟等が生じた場合、ユーザーは自己の責任と費用でこれを解決するものとし、当社は一切の責任を負いません。

## 第13条 サービスレベル

### 13.1 稼働率保証
当社は、以下の稼働率を目標として本サービスを提供します。

| 料金プラン | 稼働率保証 | 計測方法 | 補償 |
|----------|----------|---------|------|
| **Free** | 保証なし | - | なし |
| **Pro** | 99.0%/月 | Uptime監視 | SLAクレジット（返金） |
| **Team** | 99.5%/月 | Uptime監視 | SLAクレジット（返金） |

### 13.2 稼働率の計算
稼働率は、以下の計算式で算出します。

```
稼働率 = (月間総分数 - ダウンタイム分数) ÷ 月間総分数 × 100
```

### 13.3 SLA除外事項
以下の場合、SLA計算からダウンタイムを除外します。

1. 計画メンテナンス（事前通知あり）
2. 第三者サービスの障害（GitHub、Anthropic等）
3. ユーザーの責に帰すべき事由
4. 不可抗力（災害、戦争、パンデミック等）

### 13.4 SLAクレジット（返金）
稼働率が保証を下回った場合、以下の返金クレジットを提供します。

| 実績稼働率 | 返金率 |
|----------|-------|
| 99.0%未満 ~ 98.0%以上 | 10% |
| 98.0%未満 ~ 95.0%以上 | 25% |
| 95.0%未満 | 50% |

返金クレジットは、翌月の料金から自動的に差し引かれます。

### 13.5 サポート対応

| 料金プラン | サポートチャネル | 対応時間 |
|----------|---------------|---------|
| **Free** | GitHub Discussions（コミュニティ） | ベストエフォート |
| **Pro** | Email | 24時間以内（営業日） |
| **Team** | Email + Priority Queue | 12時間以内（営業日） |

### 13.6 メンテナンス通知
計画メンテナンスは、少なくとも24時間前にメールまたは本サービス内の通知で告知します。
