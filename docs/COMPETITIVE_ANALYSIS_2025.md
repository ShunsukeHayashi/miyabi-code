## 🔍 プラットフォーム詳細比較

### 1. Miyabi - AI開発自動化プラットフォーム

**開発元**: ShunsukeHayashi (日本)
**リリース**: 2025年
**ポジショニング**: 開発特化型 AI自動化プラットフォーム

#### ✅ できること (Unique Strengths)

**🎯 開発ワークフロー完全自動化**
```
Issue作成 → タスク分解 → コード生成 → テスト → レビュー → PR作成 → デプロイ
```
- **21個のAI Agents**: 7個コーディング + 14個ビジネス
  - CoordinatorAgent: タスク分解・DAG構築
  - CodeGenAgent: Claude Sonnet 4駆動のコード生成
  - ReviewAgent: 100点満点品質スコアリング
  - DeploymentAgent: Firebase/Vercel/AWS自動デプロイ
  - PRAgent: Conventional Commits準拠のPR自動作成
  - IssueAgent: 57ラベル体系による自動分類

**🐙 GitHub OS統合 (他社にない強み)**
- GitHub Projects V2: データ永続化層
- GitHub Actions: 実行エンジン
- GitHub Webhooks: イベント駆動
- 53種類の構造化ラベルシステム

**🦀 Rust実装による高性能**
- 50%以上の実行時間削減
- 30%以上のメモリ削減
- 単一バイナリ配布（Node.js不要）
- コンパイル時型安全性

**🔧 Git Worktree並列実行**
- 複数Issueの同時処理
- ブランチ分離による安全な並列作業
- 自動マージ・コンフリクト解決

**📋 1コマンドで起動**
```bash
npx miyabi init my-project  # プロジェクト作成
miyabi agent run coordinator --issue 270  # Agent実行
```

#### ❌ できないこと (Limitations)

**汎用タスク実行の弱さ**
- ❌ 電話をかける（Gensparkは可能）
- ❌ スライド作成（Gensparkは可能）
- ❌ 動画生成（Gensparkは可能）
- ❌ Webスクレイピング（Manusは可能）
- ❌ 株取引自動化（Manusは可能）

**情報検索機能**
- ❌ AI検索エンジンなし（Gensparkは専門）
- ❌ Sparkpage生成なし
- ❌ 複数LLMの統合なし

**プラットフォーム制約**
- GitHub依存度が高い（GitLab/Bitbucket未対応）
- 開発以外のドメインは弱い
- UIツール連携が限定的

**現状の制約**
- ビジネスAgent（14個）はまだ未実装（Rust移行中）
- WebUI未提供（CLIのみ）
- マーケットプレイスなし

- コンテンツ作成

**🏆 ベンチマーク性能**

**🔓 オープンソース予定**
- 2025年中に技術スタックの一部をオープンソース化予定

### 3. Genspark - AI検索エンジン + Super Agent

**開発元**: Eric Jing & Kay Zhu (元Baidu幹部)
**リリース**: 2023年
**ポジショニング**: AI検索 + タスク自動化のハイブリッド型

#### ✅ できること

**🔍 次世代AI検索 (Sparkpages)**
- 検索クエリから動的なワンページ概要を生成
- 複数ソースを統合した構造化情報
- 例：「新生児に最適な粉ミルクは？」→ 統合レポート生成

**🦸 Super Agent (2025年4月リリース)**
- **80種類以上の専門ツール**搭載
- 完全自律型ノーコードアシスタント
- 電話をかける
- スライド作成
- 動画生成
- マルチLLM統合（Mixture-of-Agents）

**📈 急成長実績**
- 200万ユーザー（2025年2月時点）
- 45日で$36M ARR達成（20人チーム）
- $530M評価額（$100M調達）

**💰 柔軟な価格体系**
- Free: 100クレジット/日
- Plus: $24.99/月（10,000クレジット）
- Pro: $249.99/月（125,000クレジット）

#### ❌ できないこと

**開発ワークフロー統合の欠如**
- ❌ GitHub統合なし
- ❌ Issue/PR管理なし
- ❌ Git操作なし
- ❌ CI/CD統合なし
- ❌ コード品質評価なし

**開発特化機能の不足**
- ❌ コード生成の深さが不足（汎用的すぎる）
- ❌ テスト自動生成なし
- ❌ 開発ワークフロー最適化なし
- ❌ ラベルシステムなし

**プラットフォーム制約**
- 検索エンジンに重点（開発は副次的）
- 開発者向け機能が浅い
- プログラミング言語特化サポートなし
