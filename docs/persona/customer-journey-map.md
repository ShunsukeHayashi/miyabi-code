# バイトプラス動画生成APIセミナー カスタマージャーニーマップ

**作成日**: 2025-10-21
**対象セミナー**: バイトプラス動画生成API販売セミナー
**対象ペルソナ**: 5個（マーケマネ、スタートアップCMO、中小企業社長、ECサイト運営者、CTO）

---

## カスタマージャーニー 5段階

```
認知（Awareness）→ 興味（Interest）→ 参加決定（Purchase）→ 購買（Purchase）→ 継続（Retention）
```

---

## ペルソナ1: 高橋（マーケティングマネージャー）- カスタマージャーニー詳細

### Stage 1: 認知段階（Awareness）

**タイミング**: セミナー開催の2-4週間前

**行動（What）**:
- LinkedIn広告で「動画制作 内製化」の広告を閲覧
- Google検索「動画マーケティング コスト削減」
- ferret、Marketing Nativeで関連記事を閲覧
- YouTubeで動画マーケティング解説動画を視聴

**思考（Think）**:
- 「動画制作コストを何とか削減したい」
- 「外注費が年々上がっていて、予算が圧迫されている」
- 「内製化は可能なのか？技術的にハードルが高そう」
- 「他社はどうやっているのだろう」

**感情（Feel）**:
- 😟 不安：内製化できるか分からない
- 😤 焦り：競合他社が動画マーケティングを強化している
- 🤔 期待：良いツールがあれば解決できるかも

**タッチポイント**:
- LinkedIn広告（スポンサード投稿）
- Google検索結果（リスティング広告）
- ferret、Marketing Native記事広告
- YouTubeのプレロール広告
- メールマガジン（マーケティング系メディア）

**ペインポイント**:
- 外注費の高騰（1本20万円〜50万円）
- 制作期間の長さ（2-4週間）
- 社内リソース不足

**チャンス（Opportunity）**:
- 「実績豊富」「簡単導入」「サポート充実」を訴求
- BtoB SaaS企業の成功事例を提示
- ROI試算ツールを提供

**推奨施策**:
1. LinkedIn広告キャンペーン（ターゲット：マーケティングマネージャー）
2. Google検索広告（キーワード：「動画制作 内製化」「動画マーケティング コスト削減」）
3. ferret記事広告（導入事例記事）
4. YouTubeプレロール広告（マーケター向け）

### Stage 2: 興味段階（Interest）

**タイミング**: セミナー開催の1-2週間前

**行動（What）**:
- セミナー案内のランディングページを熟読
- バイトプラスの企業情報、導入事例をチェック

- 過去のセミナー動画があれば視聴

**思考（Think）**:
- 「このAPIは本当に使いやすいのか？」
- 「導入後のサポートはしっかりしているか？」
- 「ROIはどれくらいで出せるのか？」
- 「競合サービスとどう違うのか？」

**感情（Feel）**:
- 🤔 興味：このサービスは良さそう
- 😐 懐疑：本当に効果があるのか？
- 📊 比較検討：他社と比べてどうか

**タッチポイント**:
- セミナーランディングページ
- 企業サイトの導入事例ページ
- YouTubeのデモ動画
- 問い合わせフォーム
- LinkedIn企業ページ
- 料金ページ

**ペインポイント**:
- 他社サービスとの明確な差別化ポイントが見えにくい
- 料金体系が不明瞭
- 導入後のサポート体制が不明

**チャンス（Opportunity）**:
- 詳細な料金シミュレーター提供
- 導入フローの可視化（ステップバイステップ）
- BtoB SaaS企業の詳細な導入事例

**推奨施策**:
1. セミナーLPの最適化（BtoB SaaS事例を前面に）
2. ROI試算ツールの提供
3. 導入事例インタビュー動画の制作
4. リターゲティング広告（LP訪問者向け）
5. 比較表の作成（競合サービスとの差別化）

### Stage 3: 参加決定段階（Purchase - セミナー参加）

**タイミング**: セミナー開催の1週間前〜当日

**行動（What）**:
- セミナー申込フォームに登録
- 事前に質問事項をメモ
- 同僚（デザイナー、エンジニア）にも参加を打診
- カレンダーに登録、リマインダー設定
- セミナー前日にリマインドメールを確認

**思考（Think）**:
- 「このセミナーで具体的な導入イメージが掴めるだろうか」
- 「技術的な質問にもその場で回答してもらえるか」
- 「同僚にも見せて、意見を聞きたい」

**感情（Feel）**:
- 😊 期待：セミナーで疑問が解消されそう
- 😬 緊張：質問できるかな

**タッチポイント**:
- セミナー申込フォーム
- 確認メール（自動配信）
- リマインドメール（開催前日）
- カレンダー招待（Googleカレンダー、Outlook）
- Zoom招待URL

**期待すること**:
1. API統合のデモ実演（実際の画面を見せてほしい）
2. 導入事例の詳細（特にBtoB SaaS企業）
3. コスト削減の具体的な試算
4. 質疑応答の時間（技術的な質問をしたい）
5. 特典や限定オファー（セミナー参加者限定の割引等）

**チャンス（Opportunity）**:
- セミナー申込完了時に「期待すること」アンケート
- 事前質問を受付（セミナーで回答）
- リマインドメールで見どころを強調

**推奨施策**:
1. 申込完了メール（自動配信）
2. リマインドメール（開催前日、開催1時間前）
3. 事前アンケート（期待すること、質問）
4. セミナー資料の一部を事前配布

### Stage 4: 購買段階（Purchase - API導入決定）

**タイミング**: セミナー終了後〜4週間以内

**行動（What）**:

- 営業担当者とのオンライン商談
- 見積書取得
- 契約書レビュー

**思考（Think）**:
- 「稟議を通すために、ROIをどう説明するか」
- 「トライアル期間で本当に使えるか検証しよう」
- 「上司を説得するための材料が必要」

**感情（Feel）**:
- 🤔 決断：導入すべきか
- 😟 不安：稟議が通るか
- 😊 期待：これで課題が解決できそう

**タッチポイント**:
- 無料トライアル申込ページ
- 営業担当者とのオンライン商談
- 見積書・契約書
- 技術サポート窓口
- 稟議書テンプレート（バイトプラス提供）
- ROI試算書（バイトプラス提供）

**決裁プロセス**:
1. **Week 1**: セミナー参加、社内共有（マネージャー）
2. **Week 2**: 稟議書作成、上司承認（CMO）
3. **Week 3**: 無料トライアル実施、技術検証
4. **Week 4**: 経理・IT部門承認
5. **Week 5**: 契約締結

**決裁者**:
- 最終決裁: CMO（Chief Marketing Officer）
- 承認必要: 経理部長、IT部長
- 推進者: マーケティングマネージャー（本人）

**チャンス（Opportunity）**:
- 稟議書テンプレート提供
- ROI試算書提供
- 導入事例集（上司向け）
- 技術検証サポート

**推奨施策**:
1. セミナー後フォローメール（翌営業日）
2. 稟議書テンプレート提供
3. ROI試算書提供
4. オンライン商談設定
5. 無料トライアル期間中のサポート強化

### Stage 5: 継続段階（Retention - 導入後）

**タイミング**: 契約締結後〜継続利用

**行動（What）**:
- API統合の実装
- マーケティングチーム向けの社内勉強会開催
- 月次レポートでROI測定
- サポートへの問い合わせ（技術的な質問）
- 新機能のテスト
- アップグレード検討

**思考（Think）**:
- 「導入効果を数値化して、上司に報告しよう」
- 「さらに活用範囲を広げられないか」
- 「このツールは本当に投資価値があった」

**感情（Feel）**:
- 😊 満足：期待通りの効果が出た
- 😌 安心：サポートがしっかりしている
- 🚀 期待：さらに活用したい

**タッチポイント**:
- 管理画面・ダッシュボード
- カスタマーサポート（チャット、メール）
- オンボーディング資料・動画
- ユーザーコミュニティ
- 定期メルマガ（活用Tips）
- 月次レビュー会（営業担当との定期ミーティング）

**導入後のサポートニーズ**:
1. **技術サポート**: API統合時のトラブルシューティング（即日対応希望）
2. **活用コンサルティング**: 動画制作の最適化アドバイス（月1回）
3. **アップデート情報**: 新機能のリリース情報（メール配信）
4. **ユーザーコミュニティ**: 他社事例の共有、ベストプラクティス
5. **研修・トレーニング**: 社内向けの操作研修（オンライン）

**ロイヤルティ要因**:
- ROI達成（外注費50%削減）
- 手厚いサポート
- 継続的な改善提案
- 新機能の早期アクセス

**推奨施策**:
1. オンボーディングプログラム（導入後1ヶ月）
2. 月次レビュー会（ROI測定、改善提案）
3. 活用Tipsメール（週1回）
4. ユーザーコミュニティ運営（Slack）
5. 新機能のベータテスター募集
6. アップグレード提案（適切なタイミング）
