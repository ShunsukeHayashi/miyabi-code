**作成日**: 2025-11-21

## 概要

このディレクトリには、電気工事業の各部署における業務フローを描写したPlantUMLファイルが含まれています。Swimlane（泳道）図を使用して、部署間の連携、情報フロー、プロセスを可視化しています。

## PlantUML の特徴

### テーマ
```plantuml
!theme cerulean-outline
```
- **cerulean-outline**: 専門的で洗練されたブルーベースのテーマ
- 背景色がリーン（白系）で、内容に集中しやすい
- 線の色が落ち着いた青で統一された配色

### Swimlane（泳道）設定
```plantuml
skinparam swimlane {
    BorderColor #004687        // 深い青
    BackgroundColor #F0F4F8    // 薄いブルー
    TitleBackgroundColor #004687
    TitleFontColor #FFFFFF
}
```

各部署ごとにカラーコード（#XXXXXX）を設定し、視覚的に区別しやすく設計：
- 営業部: E8F1F8（薄い青）
- 設計部: E8F4F1（薄い青緑）
- 施工管理部: F1F8E8（薄い黄緑）
- 現場作業部: F8F1E8（薄いベージュ）
- 安全管理部: F8E8E8（薄いピンク）
- 経理部: F8E8F1（薄い紫）
- 総務部: F1E8F8（薄い紫青）

### フロー表現方法

**基本フロー**:
```plantuml
start
: アクティビティ;
: 次のアクティビティ;
stop
```

**条件分岐**:
```plantuml
if (条件?) then (YES)
    : 処理A;
else (NO)
    : 処理B;
endif
```

**ループ処理**:
```plantuml
loop 期間
    : 繰り返し処理;
end
```

**アクター間のデータフロー**:
```plantuml
営業担当 --> 営業企画 : データ内容
```

## 使用方法

### PlantUML のコンパイル

1. **コマンドライン実行**:
```bash
plantuml industry_electrical_全体連携.puml
```

2. **VS Code での使用**:
   - PlantUML 拡張機能をインストール
   - ファイル右クリック → "Preview Current Diagram"

3. **オンラインエディタ**:
   - https://www.plantuml.com/plantuml/uml/
   - ファイル内容をコピー&ペーストして表示

### 出力形式

PlantUML は以下の形式で出力可能：
- PNG（ビットマップ画像）
- SVG（ベクター画像）
- PDF（ドキュメント）
- HTML（ウェブページ）

```bash
# PNG 出力
plantuml -tpng industry_electrical_営業部.puml

# SVG 出力
plantuml -tsvg industry_electrical_営業部.puml
```

## 業務フロー の特徴

### 電気工事業の特性

1. **受注型製造業**
   - 営業→受注→設計→施工→竣工という一連のプロセス
   - 顧客要件の詳細確認が重要

2. **安全管理の重要性**
   - 高い施工リスク（高所作業、電気関連）
   - 安全管理部が全部署と連携

3. **複数の資格要件**
   - 電気工事士、認定電気工事従事者など
   - 総務部での資格管理が重要

4. **現場特性への対応**
   - 建築現場との連携
   - リアルタイム進捗管理が必要
   - 環境変化への対応

5. **法令遵守**
   - 建設業法対応
   - 労働基準法対応
   - 電気関連法規対応

## カスタマイズガイド

### 部署の追加
```plantuml
swimlane 新部署名 #CCCCCC
    : 処理1;
    : 処理2;
end swimlane
```

### プロセスの追加
```plantuml
: **タイトル**
- 箇条書き1
- 箇条書き2;
```

### 部署間連携の追加
```plantuml
部署A --> 部署B : データ内容
```
