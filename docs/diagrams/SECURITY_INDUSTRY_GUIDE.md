## 📋 概要

警備業の組織構造、各部署の業務フロー、部署間連携を包括的に可視化したPlantUML図セットです。

### 対象業界

**警備業（Security Services Industry）**
- 常駐警備
- 機械警備（ALSシステム、監視カメラ）
- 巡回警備
- 特別警備（イベント警備、VIP警護）
- 企業警備

### 対象組織規模

- 中堅警備会社（従業員100-500名）
- 地域密着型警備企業
- 複数営業所を持つ警備企業

### 2. 営業部 業務フロー

**ファイル**: `industry_security_営業部.puml`
**行数**: 144行
**時間軸**: 案件発生～契約更新（数週間～数ヶ月）

#### 主要プロセス

```
リード獲得
  ↓
新規開拓 ← → 既存顧客管理
  ↓
ニーズ把握・現地調査
  ↓
警備計画立案（管制部と連携）
  ↓
提案書作成・見積
  ↓
提案プレゼン（競合状況判定）
  ↓
受注/失注判定
  ├→ 受注: 契約締結 → 勤務開始準備
  └→ 失注: 分析 → フォロー継続
  ↓
定期訪問・顧客フォロー
  ↓
契約更新手続
```

#### 分岐判定ポイント

| 判定 | Yes | No |
|-----|-----|-----|
| 商談機会あり？ | 初回面談 | ナーチャリング |
| 競合あり？ | 差別化説明・交渉 | - |
| 受注？ | 契約締結 | 失注分析 |
| 追加案件？ | 追加提案 | 定期訪問継続 |
| 契約更新時期？ | 更新提案 | - |

#### ステークホルダー

- **営業パーソン**: リード獲得、提案、契約
- **管制部**: 警備計画案作成
- **総務部**: 法務レビュー
- **経理部**: 契約情報連携

#### リード獲得チャネル

- Web問い合わせ
- 展示会・セミナー
- 営業紹介
- アウトバウンド営業
- 既存顧客からの紹介

### 3. 管制部 業務フロー

**ファイル**: `industry_security_管制部.puml`
**行数**: 168行
**時間軸**: 24時間運用（交代シフト制）

#### 運用体系

```
朝礼・勤務予定確認
  ├→ 常駐警備現場管理
  ├→ 巡回警備管理
  ├→ 機械警備監視（24時間）
  └→ イベント警備管理
    ↓
問題発生時エスカレーション
  ├→ 異常検知 → 警備員出動
  ├→ アラーム受信 → 現場確認 → 対応指示
  └→ 緊急事案 → 警察・消防通報
    ↓
日報作成・実績報告
```

#### 監視対象

1. **常駐警備**: 配置人員、勤務状況、現場トラブル
2. **巡回警備**: ルート確認、報告書内容、異常検知
3. **機械警備**: センサー動作、アラーム履歴、通信状態
4. **イベント警備**: 要員配置、現場対応、セキュリティ

#### アラーム対応フロー

```
アラーム受信
  ↓
内容確認（センサー種別、発動場所）
  ↓
顧客連絡（誤作動確認）
  ↓
現場出動判定
  ├→ 必要: 警備員出動 → 現地確認 → 対応
  └→ 不要: 記録のみ
  ↓
警察・消防通報判定
  ├→ 必要: 緊急通報
  └→ 不要: ログ記録
```

#### スタッフ構成

- 管制センター長
- 管制オペレーター（複数人）
- パトロール車手配担当
- 情報処理担当

### 4. 常駐警備部 業務フロー

**ファイル**: `industry_security_常駐警備部.puml`
**行数**: 174行
**時間軸**: 1勤務シフト（8-12時間）

#### 日常勤務フロー

```
出勤・朝礼
  ↓
警備品準備・施設確認
  ├→ 日中警備
  ├→ 夜間警備
  └→ 特別警備（イベント等）
    ↓
来訪者管理
  ├→ 本人確認
  ├→ 訪問目的確認
  └→ 入館可否判定
    ↓
異常検知対応
  ├→ 盗難・侵入 → 管制部報告
  ├→ 火災 → 消防通報
  ├→ 設備故障 → 報告
  └→ 人身事故 → 応急対応
    ↓
日常業務（設備操作、清掃）
  ↓
勤務引継ぎ・報告書作成
  ↓
退勤
```

#### 巡回パトロール内容

- 出入口確認（施錠状況）
- 通路・廊下確認
- 駐車場確認
- 設備動作確認
- 不審者有無確認
- 火災・災害兆候確認

#### 来訪者管理ポイント

1. **確認項目**
   - 身分証明確認
   - 訪問目的確認
   - 訪問先への事前確認
   - 滞在時間確認

2. **拒否理由**
   - 事前通知なし
   - 立ち入り禁止エリア
   - 営業・勧誘行為
   - 不審な行動

#### 異常対応ホットライン

常駐警備員は以下の異常を即座に管制部に報告:
- 侵入・盗難疑い
- 火災・煙
- ガス臭
- 設備故障
- 人身事故
- 異常な物音

### 5. 機械警備部 業務フロー

**ファイル**: `industry_security_機械警備部.puml`
**行数**: 208行
**時間軸**: 24時間監視運用 + オンサイト保守

#### 運用体系

```
監視センター業務（24時間）
  ├→ システム起動・確認
  ├→ センサー動作確認
  ├→ アラーム監視
  │   ├→ アラーム受信
  │   ├→ 内容確認
  │   ├→ 顧客確認
  │   └→ 現場対応指示
  └→ ログ記録・日報

保守点検業務
  ├→ スケジュール確認
  ├→ 顧客調整
  ├→ 現地訪問
  ├→ 装置点検
  ├→ 清掃・メンテナンス
  ├→ 動作試験
  └→ 報告書作成

新規工事業務
  ├→ 工事計画作成
  ├→ 施設調査
  ├→ 部材手配
  ├→ 工事実施
  ├→ 動作確認
  └→ 顧客研修

トラブル対応
  ├→ 遠隔診断
  ├→ 遠隔修復 or 現地修復
  └→ 原因分析・報告
```

#### 監視装置種別

1. **侵入検知システム**
   - 磁気センサー（窓・扉）
   - PIRセンサー（赤外線）
   - 振動センサー

2. **火災検知システム**
   - 煙検知器
   - 熱検知器
   - 炎検知器

3. **通信・表示装置**
   - 警報盤
   - 無線装置
   - 表示灯

#### 保守点検スケジュール

| 頻度 | 内容 | 所要時間 |
|-----|------|---------|
| 月1回 | 定期点検 | 30-60分 |
| 年2回 | 詳細点検 | 2時間 |
| 年1回 | 法定点検 | 3時間 |
| 不定期 | 故障対応 | 状況次第 |

#### トラブル対応判定

```
トラブル報告
  ↓
遠隔診断
  ├→ 修復可能 → 遠隔修復 → 完了
  ├→ 部品交換必要 → 部品手配 → 現地修復
  └→ 複雑問題 → 現地出動 → 詳細診断 → 修復
```

  ├→ 合格 → 本配置

```
日々の業務
  ├→ 売上管理
  │   ├→ 請求書作成
  │   ├→ 売上計上
  │   └→ 売上記録
  ├→ 給与・勤怠管理
  │   ├→ 勤務実績集計
  │   ├→ 給与計算
  │   ├→ 控除額計算
  │   └→ 給与支払
  ├→ 支払管理
  │   ├→ 請求書受取
  │   ├→ 内容確認
  │   ├→ 支払承認
  │   └→ 振込処理
  └→ 原価管理
      ├→ 原価計算
      └→ 原価分析

月次業務
  ├→ 売上集計
  ├→ 経費集計
  ├→ 利益計算
  ├→ 月次試算表作成
  └→ 経営層報告

四半期・年次
  ├→ 決算準備
  ├→ 決算審査
  ├→ 監査対応
  └→ 税務申告
```

#### 請求書管理

| 警備種別 | 請求項目 | 計算方法 |
|---------|---------|---------|
| 常駐警備 | 人員数 × 勤務時間 × 単価 | 基本 |
| 機械警備 | 月額基本料 + 出動料 | 固定＋変動 |
| 巡回警備 | 巡回回数 × 単価 | 回数ベース |
| イベント警備 | 要員数 × 時間 × 単価 | 実績ベース |

#### 給与計算項目

**基本給 = 基本給 + 各種手当**

| 手当名 | 計算対象 | 金額例 |
|-------|---------|--------|
| 深夜手当 | 22時-6時勤務 | 時給 × 0.5 |
| 休日手当 | 日曜・祝日勤務 | 時給 × 0.35 |
| 超過勤務 | 8時間超過分 | 時給 × 0.25-1.0 |
| 資格手当 | 保有資格 | 月3,000-10,000円 |
| 職務手当 | 職位別 | 月5,000-50,000円 |

#### 原価管理フォーカス

```
標準原価設定
  ↓
実績原価計算
  ├→ 常駐警備原価（人件費 + 装備）
  ├→ 機械警備原価（システム保守 + 人件費）
  ├→ 巡回警備原価（車両 + 人件費）
  └→ 特別警備原価（特殊要員 + 装備）
  ↓
原価分析
  ├→ 部署別原価
  ├→ 顧客別原価
  ├→ 警備種別原価
  └→ 原価乖離分析
  ↓
改善提案
```

## 🎯 各図の使用シーン

## 📈 図の共通特性

### フォーマット

- **形式**: PlantUML (.puml)
- **テーマ**: cerulean-outline
- **言語**: 日本語
- **レイアウト**: swimlane（営業部以降）または rectangle（全体連携）

### 表現方法

#### 一般的な要素

```
|部署名|         ← swimlane（水平線で部署を区切る）
start             ← 開始
:処理内容:        ← 標準処理
if (判定条件?) then (Yes)
  :Yes時処理:
else (No)
  :No時処理:
endif             ← 条件分岐
note right        ← 補足説明
fork             ← 並列処理開始
  :処理1:
fork again
  :処理2:
end fork         ← 並列処理終了
stop              ← 終了
```

#### 補足説明（note）

各ステップに対して、以下の情報をnoteで記載:
- **対象**: 具体的な対象者・物品
- **確認項目**: チェックすべき事項
- **実施内容**: 具体的な行動
- **判定基準**: 判定の基準

## 🔧 カスタマイズガイド

### 図を自組織に合わせて修正する場合

1. **部署名の変更**
   ```
   rectangle "営業部" as sales
   ↓
   rectangle "営業課" as sales （課制の場合）
   ```

2. **プロセスの追加・削除**
   - 組織固有のプロセスを追加
   - 不要なステップを削除
   - フローを簡略化

3. **スイムレーン（部署）の追加**
   ```
   |部署名|
   :処理:
   ```

4. **判定分岐の修正**
   ```
   if (条件) then (Yes)
     :Yes処理:
   else (No)
     :No処理:
   endif
   ```

### PlantUMLの編集ツール

- **VSCode + PlantUML拡張** (推奨)
- **PlantUML Online Editor** (https://www.planttext.com/)
- **PlantUML Web Server** (http://www.plantuml.com/plantuml/uml/)
