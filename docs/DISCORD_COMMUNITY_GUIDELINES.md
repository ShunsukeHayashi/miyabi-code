## コミュニティルール

### 📜 基本ルール（10項目）

#### 1. 敬意を持った対話

すべてのメンバーに敬意を持って接してください。意見の相違は自然ですが、建設的な議論を心がけましょう。

**OK例**:
- 「その実装方法もありますね。ただ、こういう懸念点があります」
- 「質問ありがとうございます。こちらのドキュメントが参考になりますよ」

**NG例**:
- 「そんな質問するなんて、初心者ですね（笑）」
- 「その実装は間違ってる。やり直せ」

---

#### 2. スパム・宣伝の禁止

関連性のない宣伝、スパム、繰り返し投稿は禁止です。Miyabiに関連する外部リンク共有は歓迎しますが、自己宣伝は控えめに。

**OK例**:
- Miyabiを使ったプロジェクトの紹介（`#showcase-projects`）
- 関連技術記事のシェア（`#links-resources`）

**NG例**:
- 無関係な商品・サービスの宣伝
- 同じメッセージの繰り返し投稿
- DMでの勧誘

---

#### 3. 適切なチャンネル使用

各チャンネルの目的に沿った投稿をお願いします。迷った場合は `#general` または `#help-general` をご利用ください。

**チャンネルガイド**:
- 技術的な質問 → `#help-general`, `#help-troubleshooting`
- Agent固有の質問 → 各Agentチャンネル（例: `#つくるん-codegen`）
- プロジェクト紹介 → `#showcase-projects`
- 雑談 → `#general`, `#off-topic`

---

#### 4. 個人情報の保護

自分や他人の個人情報（メールアドレス、住所、電話番号等）を公開しないでください。GitHubアカウント名、DiscordユーザーIDは問題ありません。

**保護すべき情報**:
- メールアドレス
- 電話番号
- 住所、勤務先詳細
- APIキー、トークン、パスワード

---

#### 5. 違法・不適切なコンテンツの禁止

違法なコンテンツ、アダルトコンテンツ、ヘイトスピーチ、差別的表現は禁止です。

**禁止コンテンツ**:
- 違法行為の助長
- アダルトコンテンツ
- ヘイトスピーチ、差別的表現
- 暴力的・グロテスクなコンテンツ

---

#### 6. 日本語・英語推奨

主要言語は日本語と英語です。他言語での投稿も歓迎しますが、英語併記を推奨します。

---

#### 7. 質問のベストプラクティス

質問する際は、以下の情報を含めると回答が得やすくなります：
- 使用しているMiyabiのバージョン
- エラーメッセージ（あれば）
- 試したこと
- 期待する動作

**良い質問例**:
```
Miyabi v0.5.0を使っています。
CodeGenAgentで以下のエラーが出ました：
[エラーメッセージ]

以下を試しましたが解決しませんでした：
- cargo clean && cargo build
- .miyabi.ymlの確認

期待する動作：
Issue #123からコード生成ができること
```

---

#### 8. 回答のベストプラクティス

回答する際は、以下を心がけましょう：
- 分かりやすく、具体的に
- 参考リンクを添える
- 「〜かもしれません」「〜を試してみてください」など、柔らかい表現
- 解決したら「解決済み」リアクション（✅）

---

#### 9. 建設的なフィードバック

批判的なフィードバックは歓迎しますが、建設的であることが重要です。代替案や改善提案を含めましょう。

**OK例**:
- 「この機能は使いにくいです。〜のような改善があれば嬉しいです」
- 「バグを見つけました。再現手順は以下の通りです」

**NG例**:
- 「このツール最悪。使えない」
- 「開発者は何もわかってない」

---

#### 10. 楽しむこと

コミュニティは学びと交流の場です。楽しんで参加してください！

## 行動規範（Code of Conduct）

### 🤝 Contributor Covenant準拠

Miyabiコミュニティは、[Contributor Covenant v2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/)に準拠した行動規範を採用しています。

### 私たちの約束

私たちはオープンで友好的なコミュニティを作ることを約束します。年齢、体型、障害、民族、性的特徴、ジェンダーアイデンティティ、経験レベル、教育、社会経済的地位、国籍、外見、人種、宗教、性的指向に関わらず、すべての人がハラスメントなく参加できる環境を提供します。

### 期待される行動

コミュニティメンバーに期待される行動：

- 他者への共感と親切さを示す
- 異なる意見、視点、経験を尊重する
- 建設的なフィードバックを与え、受け入れる
- 自分の間違いを認め、影響を受けた人に謝罪し、経験から学ぶ
- コミュニティ全体にとって最善のことに焦点を当てる

### 許容されない行動

以下の行動は許容されません：

- 性的な言葉や画像、性的な注意やアプローチ
- トローリング、侮辱的・軽蔑的なコメント、個人攻撃や政治的攻撃
- 公的または私的なハラスメント
- 明示的な許可なく、住所やメールアドレスなどの個人情報を公開すること
- プロフェッショナルな環境で不適切と見なされるその他の行為

### 執行責任

コミュニティリーダー（`@Admin`, `@Moderator`）は、この行動規範を明確にし、執行する責任があります。不適切、脅威的、攻撃的、または有害と判断される行動に対して、適切かつ公正な是正措置を取ります。

### 適用範囲

この行動規範は、以下の場合に適用されます：
- コミュニティスペース内（Discord、GitHub）
- 公的スペースで個人がコミュニティを代表している場合

### 報告

行動規範違反を目撃または経験した場合は、モデレーターに報告してください。すべての報告は迅速かつ公正に審査されます。

報告方法：
- Discord: `@Moderator` にメンション、または `#feedback` に投稿
- Email: （将来的に設定）

## FAQ（よくある質問）

### 🔰 初心者向け

#### Q1. Miyabiとは何ですか？

**A**: Miyabiは、完全自律型AI開発オペレーションプラットフォームです。GitHub as OSアーキテクチャに基づき、Issue作成からコード実装、PR作成、デプロイまでを完全自動化します。

**詳細**: [GitHub Repository](https://github.com/ShunsukeHayashi/Miyabi)

---

#### Q2. どのように始めればいいですか？

**A**: 以下のステップで始められます：

1. **インストール**:
   ```bash
   cargo install miyabi-cli
   # または
   brew install miyabi
   ```

2. **新規プロジェクト作成**:
   ```bash
   miyabi init my-project
   ```

3. **クイックスタートガイド参照**:
   [QUICK_START.md](https://github.com/ShunsukeHayashi/Miyabi/blob/main/.claude/QUICK_START.md)

---

#### Q3. Rustの経験がなくても使えますか？

**A**: はい、使えます！Miyabiは既存のプロジェクトにも追加可能で、Rustの知識は必須ではありません。ただし、Rust経験があると、より深くカスタマイズできます。

**学習リソース**:
- [The Rust Book (日本語)](https://doc.rust-jp.rs/book-ja/)
- `#links-resources` チャンネル

---

#### Q4. 「しきるん」「つくるん」って何ですか？

**A**: Miyabiの21個のAgentにつけられた、親しみやすいキャラクター名です。技術的な名前（CoordinatorAgent等）の代わりに、覚えやすい名前で呼び出せます。

**例**:
- しきるん = CoordinatorAgent（タスク分解・統括）
- つくるん = CodeGenAgent（コード生成）
- めだまん = ReviewAgent（品質レビュー）

**詳細**: [AGENT_CHARACTERS.md](https://github.com/ShunsukeHayashi/Miyabi/blob/main/.claude/agents/AGENT_CHARACTERS.md)

---

### 🔧 技術的な質問

#### Q5. どのプログラミング言語に対応していますか？

**A**: Miyabi本体はRust 2021 Editionで実装されていますが、CodeGenAgentは以下の言語でコード生成可能です：
- Rust
- TypeScript/JavaScript
- Python
- Go
- その他（実験的）

---

#### Q6. Worktree並列実行とは何ですか？

**A**: Git Worktreeを使って、複数のIssueを独立したディレクトリで同時処理する機能です。これにより、コンフリクトを最小化しながら高速な並列実行が可能になります。

**詳細**: [WORKTREE_PROTOCOL.md](https://github.com/ShunsukeHayashi/Miyabi/blob/main/docs/WORKTREE_PROTOCOL.md)

---

#### Q7. GitHub Actionsとの連携は必須ですか？

**A**: 必須ではありませんが、推奨です。GitHub Actionsを使うことで、Issue作成時の自動Agent実行やデプロイ自動化が可能になります。

---

#### Q8. 既存のプロジェクトに追加できますか？

**A**: はい、可能です。以下のコマンドで既存プロジェクトにMiyabiを追加できます：

```bash
cd existing-project
miyabi install
```

---

### 💼 Business Agent関連

#### Q9. Business Agentとは何ですか？

**A**: Miyabiには、開発だけでなくビジネス戦略、マーケティング、営業に特化した14個のBusiness Agentがあります。これにより、SaaS起業やプロダクトローンチの戦略立案も可能です。

**例**:
- AIEntrepreneurAgent: ビジネスプラン作成
- MarketingAgent: マーケティング戦略立案
- SalesAgent: セールスプロセス最適化

---

#### Q10. Coding AgentとBusiness Agentの違いは？

**A**:
- **Coding Agents（7個）**: コード生成、レビュー、デプロイ等の開発運用に特化
- **Business Agents（14個）**: 市場調査、戦略立案、マーケティング等のビジネスに特化

両方を組み合わせることで、プロダクト開発からビジネス成長まで一貫してサポートします。

---

### 🆘 トラブルシューティング

#### Q11. エラーが出ました。どうすればいいですか？

**A**: 以下の手順で解決を試みてください：

1. **エラーメッセージを確認**: 具体的なエラーメッセージをコピー
2. **ドキュメント検索**: [TROUBLESHOOTING.md](https://github.com/ShunsukeHayashi/Miyabi/blob/main/.claude/TROUBLESHOOTING.md)
3. **FAQ確認**: `#faq` チャンネル
4. **質問投稿**: `#help-troubleshooting` にエラーメッセージと状況を投稿

---

#### Q12. コミュニティに質問しても回答がない場合は？

**A**: 以下を確認してください：
- 適切なチャンネルに投稿されているか
- 質問に必要な情報（バージョン、エラーメッセージ等）が含まれているか
- 24時間以上経過しているか

それでも回答がない場合は、`@Moderator` にメンションしてください。

---

#### Q13. バグを見つけました。どうすればいいですか？

**A**: 以下の手順でバグレポートをお願いします：

1. **Discord**: `#bug-reports` フォーラムチャンネルに投稿
2. **GitHub**: [Issueを作成](https://github.com/ShunsukeHayashi/Miyabi/issues/new?template=bug_report.md)

バグレポートには以下を含めてください：
- Miyabiバージョン
- 再現手順
- 期待する動作
- 実際の動作
- エラーメッセージ

---

### 🎉 コミュニティ参加

#### Q14. コントリビュートしたいです。どうすればいいですか？

**A**: 以下の方法でコントリビュートできます：

1. **コード**: [CONTRIBUTING.md](https://github.com/ShunsukeHayashi/Miyabi/blob/main/CONTRIBUTING.md)を参照
2. **ドキュメント**: タイポ修正、翻訳、チュートリアル作成
3. **コミュニティ**: 他のメンバーの質問に回答
4. **フィードバック**: バグレポート、フィーチャーリクエスト

詳細は `#contributions` チャンネルへ。

---

#### Q15. このコミュニティのゴールは何ですか？

**A**: 以下の3つがゴールです：
1. Miyabiユーザー同士の学び合いと交流
2. プロジェクトの継続的改善（バグ修正、機能追加）
3. AI駆動開発の知識共有とベストプラクティス蓄積

## 報告・通報の方法

### 🚨 報告が必要な行動

以下の行動を目撃または経験した場合は、速やかに報告してください：

- 行動規範違反
- ハラスメント
- スパム・宣伝
- 不適切なコンテンツ
- その他、コミュニティに悪影響を及ぼす行為

### 報告方法

#### 1. Discordでの報告

**方法A: モデレーターにメンション**
- 該当メッセージに返信して `@Moderator` をメンション
- 簡単な理由を添える

**方法B: DM（ダイレクトメッセージ）**
- モデレーターにDMを送信
- 該当メッセージのリンクと理由を添える

**方法C: `#feedback` チャンネル**
- `#feedback` に投稿（匿名性が低い）
- 公開されても問題ない内容のみ

#### 2. Discord標準の報告機能

- メッセージを右クリック → 「メッセージを報告」
- Discord運営にも通知されます

### 報告時に含めるべき情報

- **誰が**: 問題行動を起こしたユーザー
- **何を**: 具体的な行動（メッセージリンク推奨）
- **いつ**: タイムスタンプ（メッセージから自動取得）
- **どこで**: チャンネル名
- **なぜ問題か**: ルール違反の理由

### 報告後の流れ

1. **受理**: モデレーターが報告を受理（24時間以内）
2. **調査**: 状況を確認
3. **対応**: 適切な措置を決定
4. **通知**: 報告者に結果を通知（プライバシー保護のため詳細は非公開）

## モデレーションポリシー

### ⚖️ 違反時の措置

違反の重大度に応じて、以下の措置を取ります：

#### レベル1: 軽微な違反（初回・軽度）

**違反例**:
- 不適切なチャンネルへの投稿
- 軽度のスパム
- トピックからの逸脱

**措置**:
- ⚠️ **警告**: モデレーターからの注意喚起
- メッセージの削除（場合により）
- ルール再読のお願い

**再発時**: レベル2へエスカレーション

---

#### レベル2: 中程度の違反（再発・中度）

**違反例**:
- 繰り返しのスパム
- 軽度の不適切な表現
- 他メンバーへの軽度の攻撃

**措置**:
- ⏸️ **タイムアウト**: 24時間〜7日間の投稿禁止
- 公式警告の記録
- モデレーターとの個別対話

**再発時**: レベル3へエスカレーション

---

#### レベル3: 重大な違反（悪質・重度）

**違反例**:
- 行動規範の重大な違反
- ハラスメント
- 差別的表現
- 個人情報の無断公開

**措置**:
- 🚫 **BAN（永久追放）**: サーバーからの永久追放
- Discord運営への報告（必要に応じて）
- 関連アカウントのBAN（場合により）

**再発時**: 措置の即時執行（猶予なし）

---

### 異議申し立て

措置に異議がある場合は、以下の方法で申し立てができます：

1. **申し立て先**: `@Admin` にDM
2. **期限**: 措置から7日以内
3. **審査**: Admin + Moderatorで再審査
4. **結果**: 7日以内に通知
