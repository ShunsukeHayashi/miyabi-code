## 🎯 重要決定事項

### 1. 社名・サービス名決定

#### 最終決定
- **社名**: **株式会社世界の始まり**
- **サービス名**: **世界** または **始まり**
- **コンセプト**:
  - 「始まり」= 理論・概念（オメガ理論）
  - 「世界」= プロダクト・サービス

#### 決定理由
1. **ストーリー性**: 「始まりが世界を作った」という神話的コンセプト
2. **ポジティブ**: 新しいことが始まる、未来志向
3. **商標取得可能性**: 第9類・第42類で取得可能性が高い
4. **ブランド力**: 「世界の終わり」のような認知度は期待できないが、逆にポジティブ
5. **説明しやすい**: 日本人なら誰でも意味が理解できる
6. **SDGsとの親和性**: 「持続可能な世界を作る」というストーリーと整合

#### 商標登録戦略
- **優先順位1**: 「株式会社世界の始まり」（社名）
- **優先順位2**: 「世界」（サービス名）
- **優先順位3**: 「始まり」（理論名・サブブランド）
- **対象区分**:
  - **第9類**: 電子計算用プログラム、ダウンロード可能なコンピューターソフトウェア
  - **第42類**: コンピュータープログラムの設計・作成、SaaS提供

### 2. ビジネスモデル

#### 3つの収益モデル

##### モデル1: BtoC サブスクリプション（月額課金）
- **ターゲット**: 一般コンシューマー
- **価格帯**: 月額1万円程度
- **特徴**:
  - ChatGPT型のWebサービス
  - 爆発的な広がりの可能性
  - マネタイズは緩やか
  - 時価総額向上に寄与
- **優先度**: 低（初期は見送り）

##### モデル2: BtoB 大手企業向けカスタマイズ開発
- **ターゲット**: 業界トップ企業
- **価格帯**: 1億〜3億円 / プロジェクト
- **特徴**:
  - 完全カスタマイズ開発
  - 雅（Miyabi）を活用した超高速開発
  - 業界初事例として実績化
  - 以降、同業界向けパッケージ化
- **優先度**: 最高（年内に10社契約目標）

##### モデル3: BtoB パッケージ SaaS
- **ターゲット**: 中堅〜中小企業
- **価格帯**:
  - 月額1万円/店舗 × 店舗数
  - または月額100万円〜/企業
- **特徴**:
  - モデル2で開発したものをパッケージ化
  - 業界特化型SaaS
  - 継続収入型
- **優先度**: 中（モデル2の後に展開）

#### 収益モデルの組み合わせ例

**パターンA: 初期費用 + 月額課金**
```
初期費用: 2億円（カスタマイズ開発）
月額料金: 月1,000万円（保守・運用）
契約期間: 3年〜5年
```

**パターンB: 成果報酬型**
```
初期費用: 500万円（格安）
成果報酬: 売上の10%
契約期間: 5年
```

**パターンC: サブスクリプション**
```
初期費用: なし
月額料金: 1万円/店舗 × 1,000店舗 = 月1,000万円
契約期間: 1年（自動更新）
```

### 5. 組織体制

#### 新会社（株式会社世界の始まり）の体制

**必要人員**:
1. **プリセールス/コンサルタント**: 2〜3名
   - 要件定義前の「要望定義」
   - 顧客との壁打ち
   - ソリューション設計

2. **技術者（エンジニア）**: 3〜5名
   - 雅を使った開発実行
   - プロダクト実装
   - デモ作成

3. **営業（リード獲得）**: ネクシーズから派遣
   - 大手企業のリード獲得のみ
   - 時価総額100億円以上/上場企業のみ
   - 小規模案件は対象外

4. **保守運用**: パートナー企業 or ネクシーズ
   - 新会社では保守を抱えない
   - 外部委託で対応

#### ネクシーズとの役割分担

| 役割 | 担当 | 詳細 |
|------|------|------|
| 大型案件獲得 | 新会社 + ネクシーズ | 共同営業 |
| カスタマイズ開発 | 新会社 | 雅を活用した開発 |
| 中小案件開発 | ネクシーズ | 雅を新会社からライセンス |
| 保守運用 | ネクシーズ or パートナー | 外部委託 |
| リード獲得 | ネクシーズ営業 | 条件付きリード獲得 |

### 6. マーケティング・営業戦略

#### 営業アプローチ

##### ステップ1: トップダウン営業（大手企業）

**対象**: 業界トップ企業の経営者・CxO

**手法**:
1. 既存の人脈を活用（近藤社長、高木氏等）
2. 「業界初のAI活用事例を作りませんか？」
3. 先行者メリットを強調
4. 「完成してから見せる」スタイル
   - 要望をヒアリング
   - デモを作成（無料）
   - 気に入ったら契約

**価格交渉**:
- 「初期費用 + 継続収入」または「成果報酬」を提示
- 「先行者特典」として割引を提示
- パッケージ化後の安価版を気にさせない説明

##### ステップ2: 実績ベース営業（中堅企業）

**対象**: 同業界の2番手〜10番手企業

**手法**:
1. 「業界トップ企業が導入しました」
2. パッケージ版を提示（価格明確）
3. ネクシーズ営業部隊が展開

**価格設定**:
- 月額1万円/店舗 × 店舗数
- または月額100万円/企業

##### ステップ3: 政府機関営業

**対象**: 各省庁（デジタル庁、財務省、文科省、厚労省等）

**手法**:
1. 高木氏経由で事務官にアポ
2. 省庁別の企画を提案
3. 随意契約を狙う（上流を縛る）
4. 補助金制度の企画提案

**タイムライン**:
- 企画提案〜決定: 3ヶ月
- 本格導入: 6ヶ月〜1年

### 8. 技術戦略

#### 雅（Miyabi）の活用

**強み**:
1. **超高速開発**: 通常数ヶ月〜1年の開発を数週間で実現
2. **ノーコード/ローコード**: エンジニア不足を解消
3. **要件定義不要**: 雅との対話で要件が自動的に固まる
4. **継続学習**: 使えば使うほど企業に最適化

**差別化ポイント**:
- ChatGPTではアプリは作れない
- 雅は「アプリを作れる」
- 顧客は雅の中で仕事をする → 要件定義が自動的に固まる

#### 開発フロー

```
従来の開発フロー:
要件定義（1ヶ月） → 設計（2ヶ月） → 開発（6ヶ月） → テスト（2ヶ月） = 11ヶ月

雅を使った開発フロー:
雅との対話（1週間） → 自動開発（1週間） → テスト（1週間） = 3週間
```

#### 技術スタック

- **フロントエンド**: React / Next.js
- **バックエンド**: Rust / Node.js
- **AI/ML**: Claude Sonnet 4, GPT-4, Gemini（ハイブリッド）
- **データベース**: PostgreSQL, Qdrant（ベクトルDB）
- **インフラ**: Firebase, GCP, AWS
- **CI/CD**: GitHub Actions

### 10. リスクと対策

#### リスク1: 期待値調整のズレ

**課題**: AI案件は期待値が膨らみやすく、ほぼ全案件で「事故る」

**対策**:
1. 「完成してから見せる」スタイル
2. デモ段階で機能を確認
3. 契約書で明確なスコープ定義
4. 先行者メリットを強調し、後発との価格差を正当化

#### リスク2: 足が重くなる（保守案件の増加）

**課題**: 受託案件が増えると保守で手が回らなくなる

**対策**:
1. 保守はネクシーズ or パートナーに外注
2. パッケージ化を早期に進める
3. 大手案件に絞る（小規模案件は受けない）

#### リスク3: 商標が取れない

**課題**: 「世界の始まり」の商標が取れない可能性

**対策**:
1. 複数の候補を並行申請（「世界」「始まり」等）
2. 最悪の場合、「世界1」等の代替案
3. 弁理士に相談し、取得可能性を事前確認

#### リスク4: 競合の出現

**課題**: 同様のAI開発支援サービスが出てくる

**対策**:
1. 業界トップ企業を早期に獲得（先行者優位）
2. 雅の技術的優位性（オメガ理論）を強調
3. 実績を積み上げてブランド力を構築
