## GitHubをOSの外枠として使う

Miyabiのアーキテクチャの核心は、**「GitHub as OS」**です。

GitHubの各機能を、OSのコンポーネントとして使います：

### GitHub機能のMiyabiでの役割

**Issues（イシュー）**
→ タスクキュー（ユーザーのTo-Doリスト）

→ タスクの状態管理（STATE, AGENT, PRIORITY等）

→ 成果物の統合とレビュー

**Actions（アクション）**
→ 実行エンジン（CI/CD自動化）

**Webhooks（ウェブフック）**
→ イベントバス（状態変化の通知）

**Discussions（ディスカッション）**
→ メッセージキュー（エージェント間通信）

**Pages（ページ）**
→ 成果物のホスティング（ドキュメント公開）

これにより、**GitHubというプラットフォームが、エージェントが動作するためのOS**として機能します。
