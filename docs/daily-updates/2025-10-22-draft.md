## 💡 技術的学び

### 1. Bashスクリプトのエラーハンドリング

**問題**: `set -e` が厳格すぎて、grep がパターンを見つけられない場合にスクリプト全体が中断

**解決**: より柔軟なエラーハンドリングに変更
```bash
# 修正前
set -e
if ! grep -q "pattern" "$FILE"; then
    # エラーで中断
fi

# 修正後
# set -e を削除
if ! grep -q "pattern" "$FILE" 2>/dev/null || [ ! -s "$FILE" ]; then
    # エラーを許容しつつ処理継続
fi
```

### 2. PlantUMLの日本語対応

**問題**: robust timing diagram や gantt chart で日本語が正しく表示されない

**解決**: シンプルな component diagram に切り替え + フォント指定
```plantuml
@startuml
!theme plain
skinparam backgroundColor white
skinparam defaultFontName "Hiragino Sans"  ← 日本語フォント
# ...
@enduml
```

### 3. JSON生成時のエスケープ処理

**問題**: Issue タイトルに引用符やバックスラッシュがあると JSON が壊れる

**解決**: jq を使った安全なJSON生成
```bash
# 修正前（危険）
ISSUES="{\"title\": \"$title\"}"  # $title に " があるとJSON破損

# 修正後（安全）
issue_json=$(jq -n \
  --arg title "$title" \
  '{title: $title}')  # jq が自動エスケープ
```
