#### Acceptable Behavior

#### Unacceptable Behavior
‚ùå **Spam/Self-Promotion**: Excessive promotional content without community value
‚ùå **Harassment**: Personal attacks, discrimination, or intimidating behavior
‚ùå **Off-Topic Discussions**: Consistently derailing conversations
‚ùå **Low-Quality Content**: Repeatedly posting low-effort or irrelevant content
‚ùå **Competitive Bashing**: Negative commentary about competing platforms
‚ùå **Privacy Violations**: Sharing private information without consent

### Content Guidelines

#### Technical Content Standards
- **Code Sharing**: Use proper formatting and include context
- **Problem Reports**: Include reproduction steps and environment details
- **Solutions**: Mark solutions clearly and explain reasoning
- **Documentation**: Link to official docs when available

#### Project Showcase Standards
- **Original Work**: Only share projects you have rights to showcase
- **Clear Description**: Explain what the project does and technologies used
- **Miyabi Integration**: Clearly show how Miyabi framework is utilized
- **Results/Metrics**: Share measurable outcomes when possible

### Moderation Policies

#### Warning System
1. **First Warning**: Private message explaining issue
2. **Second Warning**: Public warning with 24-hour timeout
3. **Third Warning**: 7-day temporary ban
4. **Final Action**: Permanent ban for severe or repeated violations

#### Immediate Action Violations
- Hate speech, harassment, or discrimination
- Spam or malicious content
- Sharing inappropriate or illegal content
- Doxxing or privacy violations
- Coordinated disruption attempts

#### Appeal Process
1. Contact moderation team via DM
2. Provide clear explanation of situation
3. 48-hour review period
4. Decision communicated with reasoning
5. Final appeal to admin team if needed

## Bot Integration & Automation

### GitHub Integration Bot

#### Features:
- **Issue Notifications**: New issues, status changes, assignments
- **PR Updates**: Pull request creation, reviews, merges
- **Release Announcements**: New releases with changelog summaries
- **Contribution Recognition**: Automated thanks for contributions
- **Status Sync**: Real-time sync between GitHub and Discord discussions

#### Configuration:
```yaml
github_integration:
  repositories:
    - miyabi-ai/core
    - miyabi-ai/dashboard
    - miyabi-ai/github-app
  notifications:
    issues: "#development-discussion"
    pull_requests: "#code-review"
    releases: "#announcements"
  mention_mapping:
    github_username: discord_user_id
```
