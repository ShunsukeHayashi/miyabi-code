#### 4. Phase 9: コミュニティフィードバック自動化システム ✨ NEW

**目的**: ユーザーフィードバックをリアルタイムに開発に反映し、コミュニティの声を製品に組み込む

**実装完了日**: 2025-10-18

##### システム概要

Discord上のユーザーフィードバックを自動的にGitHub Issueに変換し、開発チームが即座に対応できる仕組みを構築。

```
┌────────────────────────────────────────────────────────┐
│  Discord Community Feedback                             │
│  - バグ報告、機能リクエスト、質問                        │
└────────────────────┬───────────────────────────────────┘
                     │
                     ▼
┌────────────────────────────────────────────────────────┐
│  Miyabi Bot (Feedback Collector)                       │
│  - キーワードベースの自動分類                            │
│  - リアクション投票による優先度判定                      │
│  - !miyabi issue create <channel_id> <message_id>      │
└────────────────────┬───────────────────────────────────┘
                     │
                     ▼
┌────────────────────────────────────────────────────────┐
│  GitHub Issue (自動作成)                                │
│  - フィードバック内容                                    │
│  - 投稿者情報、Discord URL                              │
│  - 自動ラベル付与（type, priority, community）          │
│  - 優先度スコア                                          │
└────────────────────────────────────────────────────────┘
```

##### フィードバック分類システム

**4つのタイプ**:

1. **🐛 Bug** - バグ報告
   - キーワード: バグ、bug、エラー、error、動かない、doesn't work
   - 自動ラベル: `🐛 type:bug` + `🔥 priority:P0-Critical` (高優先度)
   - 優先度: 高

2. **✨ Feature** - 機能リクエスト
   - キーワード: 機能、feature、欲しい、want、追加、add
   - 自動ラベル: `✨ type:feature` + `📝 priority:P3-Low`
   - 優先度: 中

3. **❓ Question** - 質問
   - キーワード: 質問、question、どうやって、how to、教えて、help
   - 自動ラベル: `❓ type:question`
   - 優先度: 低

4. **💬 General** - 一般的なフィードバック
   - 上記以外のすべて
   - 自動ラベル: `💬 type:discussion`
   - 優先度: 低

**共通ラベル**:
- `👥 community` - コミュニティからのフィードバック
- `📥 state:pending` - 未処理

##### 優先度スコアリング

リアクションベースの自動優先度判定:

| リアクション | 重み | 説明 |
|------------|-----|------|
| 👍 Upvotes | ×1 | 一般的な賛同 |
| 🐛 Bug Reports | ×3 | バグ報告の重要度 |
| ✨ Feature Requests | ×2 | 機能リクエストの需要 |
| 🔥 High Priority | ×5 | 緊急度の高さ |

**スコア閾値**: 5ポイント以上で重要フィードバックと判定

**計算例**:
- 👍×3 + 🐛×2 = 3×1 + 2×3 = **9点** → 重要フィードバック ✅
- 👍×2 + ✨×1 = 2×1 + 1×2 = **4点** → 通常フィードバック
- 🔥×1 = 1×5 = **5点** → 重要フィードバック ✅

##### 使い方

**ステップ1**: フィードバックメッセージのURLを取得
```
https://discord.com/channels/1199878847466836059/1199878848968405057/1234567890123456789
                            └─ guild_id ─┘  └─ channel_id ┘  └─ message_id ┘
```

**ステップ2**: Botコマンドを実行
```
!miyabi issue create 1199878848968405057 1234567890123456789
```

**ステップ3**: 自動的にGitHub Issueが作成される！
- Issueタイトル: `[Type] フィードバック内容の先頭60文字`
- Issue本文: フィードバック詳細 + Discord情報
- ラベル: 自動推論されたラベル群
