# Lark Base 命名規則 - 完全版（識学理論準拠）

## 📖 概要

Lark Baseでは従来のRDBMSと異なり、**表示名のみ**で管理されます。物理名という概念は存在しません。
この特性を活かし、識学理論に基づいた直感的で業務担当者が理解できる命名規則を定義します。

## 🎯 基本原則（識学理論ベース）

### 重要なポイント
- ✅ すべて表示名のみで管理
- ✅ 日本語・絵文字が自由に使用可能
- ✅ 直感的で業務担当者が理解できる命名
- ✅ 番号プレフィックスによる順序制御
- ✅ 色分けによる視覚的識別

### 識学理論に基づく設計原則
1. **責任と権限の明確化** - 担当者・承認者・責任者を明確に分離
2. **結果重視の命名** - 感情的判断を排除し、客観的データを重視
3. **位置（階層）の明確化** - 組織の階層を反映した権限設定
4. **誤解・錯覚の排除** - 曖昧な表現を避け、明確な基準を設定
5. **感情的判断の排除** - 数値・事実ベースの判断材料を提供

## 📊 テーブル命名規則

### 1. 基本的な命名パターン

#### 業務単位での命名
- **管理対象を明確に**: "取引先", "協力会社", "現場管理"
- **時系列データ**: "打ち合わせ履歴", "進捗報告", "作業実績"
- **設定データ**: "商品マスター", "社員情報", "エリア設定"

#### 推奨パターン
```
シンプルな日本語名:
- "顧客管理", "案件管理", "作業履歴"

番号付きで順序制御:
- "1.顧客情報", "2.案件管理", "3.作業履歴"

絵文字で視認性向上:
- "👥顧客管理", "💼案件管理", "🔧作業履歴"
```

## 🚀 実装時のベストプラクティス

### 1. 一貫性の確保

#### 命名の統一
- 同じ意味のフィールドは同じ名前を使用
- 略語の使用ルールを事前に定義
- 敬語・丁寧語の使い分けを統一

**例**:
```
✅ 良い例: "会社名", "担当者名", "商品名"
❌ 悪い例: "会社名", "担当者", "商品"
```

### 2. 可読性の向上

#### 長さの調整
- 30文字以内を推奨
- 長すぎる場合は略語や改行を検討
- 業務担当者が理解できる用語を使用

#### 絵文字の活用
- 過度に使用しない（1フィールドに1つまで）
- 業務に関連した直感的なものを選択
- チーム内で絵文字の意味を統一

### 3. 将来の拡張性

#### 拡張を考慮した命名
- 機能追加時に既存名と矛盾しない
- 階層構造を意識した命名
- 名前の変更コストを考慮

**例**:
```
✅ 良い例: "顧客ランク", "案件ステータス", "作業進捗"
❌ 悪い例: "ランク", "ステータス", "進捗"
```

## 🎯 まとめ

Lark Baseでは表示名のみで管理されるため、以下の識学理論に基づいた命名原則を重視してください：

### 識学理論の5つの核心概念

1. **責任と権限の明確化**
   - 責任者を明確に表示: "営業責任者", "制作責任者", "承認責任者"
   - 権限レベルを明確に: "決裁権限", "承認権限", "実行権限"

2. **結果重視の評価システム**
   - 客観的数値による評価: "達成率90%", "完了件数15件"
   - 結果による分類: "目標達成", "目標未達", "大幅未達"

3. **位置（階層）の明確化**
   - 組織階層を反映: "部長承認", "課長確認", "担当者実行"
   - 階層別の責任範囲: "経営判断", "管理職承認", "一般職実行"

4. **誤解・錯覚の排除**
   - 曖昧な表現を避ける: "期限内完了", "基準値達成", "要件満足"
   - 明確な基準設定: "80点以上合格", "24時間以内対応"

5. **感情的判断の排除**
   - データに基づく判断: "数値分析結果", "客観的評価", "測定データ"
   - 感情を排除した表現: "基準達成", "要件充足", "条件クリア"

### 実装における重要ポイント

**基本的な命名原則**:
1. 直感的な日本語名 + 識学理論の責任明確化
2. 絵文字で視認性向上 + 階層の可視化
3. 番号プレフィックス + 結果重視の順序
4. 色分けで状態を視覚化 + 客観的判断支援
5. 業務担当者が理解できる用語 + 組織意識の統一

**組織運営の効率化**:
- 責任者が一目で分かる設計
- 結果が数値で確認できる仕組み
- 階層に応じた権限設定
- 感情的判断を排除した客観的データ
- 誤解を生まない明確な表現

これらの識学理論に基づいた命名規則により、Lark Baseが単なるデータベースではなく、
**組織の意識統一と効率的な運営を支援する戦略的ツール**として機能します。
